# YQP-Archwiki
**[文件目录](dotfile/File_Directory.md)**

基于 ArchWiki 官方内容，结合个人实践经验二次梳理、重构的 Arch Linux 中文指南，全部采用 Markdown 格式编写，兼顾可读性、实用性和本地化适配，适配 Obsidian/GitHub/Gitee 等平台的 Markdown 渲染与链接跳转。
## 🌟 项目特点
- **轻量化重构**：基于 ArchWiki 官方文档核心内容，剔除冗余信息，补充个人踩坑经验与本地化配置（如国内源、中文环境、常用软件适配）；
- **纯 Markdown 格式**：所有文档均为 `.md` 格式，支持 Obsidian 本地阅读、GitHub/Gitee 网页渲染，兼容主流 Markdown 编辑器；
- **结构化组织**：按「基础配置 / 软件安装 / 系统优化 / 故障排查」分类，目录层级清晰，跨文档链接可直接跳转；
- **实践验证**：所有配置、命令均经过 Arch Linux 实机验证，避免官方文档中「理论可行但实际踩坑」的问题；
- **中文友好**：全程中文编写，适配中文环境（如输入法、字体、本地化），解决 ArchWiki 英文文档的理解门槛。
## 🚀 快速开始
### 1. 获取项目
```
# 克隆仓库到本地
git clone https://gitee.com/bzhanyiqiu/YQP-Archwiki
```
### 2. 查看文档

#### 方式 1：本地编辑器查看（推荐）

- 用 **Obsidian** 打开项目根目录：自动识别跨文档链接，支持目录导航、全文搜索，适配自定义宽度 / 主题；
- 用其他 Markdown 编辑器（如 Typora/Zettlr）：直接打开 `.md` 文件即可阅读，链接跳转需确保路径正确。
#### 方式 2：网页端查看
- 访问 GitHub/Gitee 仓库：网页端原生渲染 Markdown，点击文档内的 `[链接](路径.md)` 可直接跳转到对应文档。
### 3. 使用建议
- 新手建议按「基础安装 → 系统配置 → 常用软件」的顺序阅读；
- 文档中所有命令均基于 `bash` 终端，部分配置需 `root` 权限（标注 `sudo`）；
- 关键配置步骤附带「验证命令」，执行后可确认配置生效（如 `locale -a` 验证中文环境）。
## 📝 文档规范
1. **链接格式**：跨文档链接使用相对路径（如 `[国内源替换](02-系统配置/02-国内源替换.md)`），确保本地 / 网页端均可跳转；
2. **命令格式**：所有终端命令用 ```bash 代码块包裹，关键命令标注「必执行 / 可选」；
3. **版本说明**：文档中涉及的软件版本均基于 Arch Linux 最新稳定版，定期同步 ArchWiki 更新；
4. **备注格式**：个人实践经验用「💡 备注」标注，与官方内容区分。
